Why models

  • to generalize well
  • avoiding high bias (underfitting)
  • avoiding high variance (overfitting)

Over and underfitting

  • overfitting
  • all trends captured
  • not the dominant one

  • underfitting
  • not capturing any trend
  • not the dominant one

Overfitting - 1

  • the essence of an algorithm like Linear Regression is to capture the dominant trend and fit our line within that trend
  • overfitting is the case where the overall cost is really small, but the generalization of the model is unreliable. This is due to the model learning too much from the training data set
  • learned very well from from the training data
  • can’t make reliable predictions for new inputs

Overfitting - 2

  • increase training data
  • reduce model complexity
  • early stopping during the training phase (have an eye over the loss over the training period as soon as loss begins to increase stop training)
  • Ridge Regularization and Lasso Regularization
  • use dropout for neural networks to tackle overfitting

Overfitting - 3

  • detecting:
    • good performance on the training data, poor generalization to other data

Underfitting - 1

  • just as bad for generalization of the model as overfitting
  • underfitting is the case where the model has not learned enough from the training data, resulting in low generalization and unreliable predictions
  • a simplistic line that does not generalize well

Underfitting - 2

  • increase model complexity
  • increase number of features, performing feature engineering
  • remove noise from the data.
  • increase the number of epochs or increase the duration of training to get better results

Underfitting - 3

  • detecting:
    • poor performance on the training data, poor generalization to other data

Unbalanced target variable

  • one of the common issues found in datasets that are used for classification is imbalanced classes issue
  • an unequal distribution of classes within a dataset
  • e.g. for example, in a credit card fraud detection dataset, most of the credit card transactions are not fraud and a very few classes are fraud transactions. This leaves us with something like 50:1 ratio between the fraud and non-fraud classes

What to do

  • undersampling is the process where you randomly delete some of the observations from the majority class in order to match the numbers with the minority class
  • oversampling is the process of generating synthetic data that tries to randomly generate a sample of the attributes from observations in the minority class
  • a number of methods used to oversample, e.g. SMOTE

Naive over and undersampling

  • random oversampling: randomly duplicate examples in the minority class
  • random rndersampling: randomly delete examples in the majority class
  • they assume nothing about the data and no heuristics are used
  • both techniques can be used for two-class (binary) classification problems and multi-class classification problems with one or more majority or minority classes

Naive oversampling - Python - 1

from collections import Counter
from sklearn.datasets import make_classification
from imblearn.over_sampling import RandomOverSampler
# define dataset
X, y = make_classification(n_samples=10000, weights=[0.99], flip_y=0)

print(Counter(y))
## Counter({0: 9900, 1: 100})

Naive oversampling - Python - 2

# define oversampling strategy
oversample = RandomOverSampler(sampling_strategy='minority')
# fit and apply the transform
X_over, y_over = oversample.fit_resample(X, y)
# summarize class distribution
print(Counter(y_over))
## Counter({0: 9900, 1: 9900})

Naive undersampling - Python - 1

from imblearn.under_sampling import RandomUnderSampler

X, y = make_classification(n_samples=10000, weights=[0.99], flip_y=0)

print(Counter(y))
## Counter({0: 9900, 1: 100})
undersample = RandomUnderSampler(sampling_strategy='majority')
# fit and apply the transform
X_over, y_over = undersample.fit_resample(X, y)
# summarize class distribution
print(Counter(y_over))
## Counter({0: 100, 1: 100})

SMOTE

  • Synthetic Minority Over-sampling Technique
  • creates new observations along the lines of a randomly chosen point and its nearest neighbors

SMOTE - Python

import pandas as pd
from collections import Counter
from imblearn.over_sampling import SMOTE

url="https://raw.githubusercontent.com/urosgodnov/datasets/master/Hacide_train.csv"
data=pd.read_csv(url)
data_clean=data.iloc[:,1:4]
sm = SMOTE(sampling_strategy='auto', k_neighbors=2)

X = data_clean.iloc[:, 1:3]
y = data_clean.iloc[:,0]

X_res, y_res = sm.fit_resample(X, y)

print(Counter(y_res))
## Counter({0: 980, 1: 980})
